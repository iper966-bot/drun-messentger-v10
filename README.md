# Друн мессенджер

Простой веб-мессенджер в духе Telegram: при регистрации — ник, имя
пользователя и пароль; ник можно менять когда угодно (ни на что не влияет),
по имени пользователя вас находят через поиск. После регистрации можно
сменить аватарку и ник в любой момент. Переписка — в реальном времени
(через WebSocket), данные хранятся в SQLite-базе прямо на сервере.
Работает и в браузере на компьютере, и на телефоне (адаптивный интерфейс).

## Что внутри
- `server.js` — точка входа: собирает Express-приложение и поднимает HTTP + WebSocket
- `src/` — backend, разбитый по модулям:
  - `config.js` — константы и справочники (подарки, стикеры, мини-игры, env)
  - `db.js` — инициализация SQLite и миграции схемы
  - `helpers.js` — сериализация пользователей/чатов, реакции, статусы
  - `media.js` — сохранение присланных фото/видео/голосовых на диск
  - `middleware.js` — авторизация (JWT) и права создателя
  - `ws.js` — WebSocket-сервер и рассылка событий в реальном времени
  - `routes/` — REST API по областям: `auth`, `catalog`, `users`,
    `messages`, `chats`, `games`, `owner`
- `public/` — frontend без сборки:
  - `index.html` — разметка
  - `styles.css` — стили
  - `app.js` — весь клиентский JavaScript
- `package.json` — список зависимостей
- `uploads/` — сюда складываются присланные фото и видео (создаётся сама)

## Фото и видео
- Кнопка-скрепка рядом с полем ввода — можно выбрать фото или видео.
  Работает и в личке, и в группах/каналах.
- Ещё файл можно просто перетащить в окно переписки или вставить из
  буфера через Ctrl+V.
- К файлу можно добавить подпись — просто напиши текст перед отправкой.
- Фото перед отправкой автоматически ужимается в браузере (до 1600px по
  длинной стороне, JPEG) — чтобы не гонять по сети 25 МБ с телефона.
  GIF и видео уходят как есть, без перекодирования.
- Лимит на файл — 25 МБ. Поддерживаются jpg, png, webp, gif, mp4, webm, mov.
- Клик по фото открывает его на весь экран (закрыть — Esc или клик мимо).
- **Важно:** файлы лежат на диске в папке `uploads/`, а не в базе. На
  бесплатном плане Render эта папка так же обнуляется при перезапуске,
  как и `nova.db` (см. раздел про базу ниже).

## Реакции
- Наведи курсор на сообщение (на телефоне — кнопка видна сразу) и нажми
  кружок сбоку, чтобы выбрать реакцию: 👍 ❤️ 😂 🔥 😮 😢 🍾
- Одна реакция от человека на сообщение: клик по другой — заменит,
  клик по своей же — снимет.
- Реакции прилетают собеседнику мгновенно через WebSocket.
- Ставить можно на любое сообщение — текст, фото, видео и подарок.

## Как запустить локально (для проверки)

1. Установи [Node.js](https://nodejs.org/) версии **22.5 или новее** (нужна
   для встроенной поддержки SQLite без сборки нативных модулей).
2. Если папка проекта уже открывалась раньше и `npm install` падал с
   ошибкой (например, `node-gyp`, `better-sqlite3`) — удали папку
   `node_modules` и файл `package-lock.json`, если они есть, перед
   следующим шагом.
3. Открой терминал в папке проекта и выполни:
   ```
   npm install
   npm start
   ```
4. Открой в браузере: `http://localhost:3000`
5. Чтобы друзья из твоей локальной сети тоже могли зайти — узнай свой
   локальный IP (`ipconfig` на Windows / `ifconfig` на Mac/Linux) и
   отправь им ссылку вида `http://ТВОЙ_IP:3000`.

## Как выложить в интернет (чтобы был доступен всем, не только в локальной сети)

Самый простой бесплатный вариант — **Render.com**. Подробная пошаговая
инструкция (включая настройку секретов и постоянного хранилища для
базы данных) — в файле [`DEPLOY.md`](./DEPLOY.md). Там же — Blueprint
(`render.yaml`), который настраивает всё автоматически: достаточно
залить проект в GitHub и один раз нажать "New → Blueprint" на Render.

**Важно:** на бесплатном плане Render "усыпляет" сервис после 15 минут
без запросов, и первый заход после паузы может занять 30–60 секунд —
это нормально, просто нужно подождать.

## Другие варианты хостинга
Точно так же подойдут: **Railway.app**, **Fly.io**, **Cyclic**,
или обычный VPS (DigitalOcean, Timeweb и т.п.) — команды `npm install`
и `npm start` везде одинаковые.

## Как это работает
- При регистрации нужны три поля: **ник** (отображаемое имя, можно менять
  когда угодно и ни на что это не влияет), **имя пользователя** (логин
  латиницей — по нему ищут в поиске и входят в аккаунт, изменить нельзя)
  и **пароль**.
- После входа можно в любой момент открыть профиль (клик по своей аватарке)
  и сменить ник и/или фото — это никак не связано с именем пользователя
  и паролем.
- Аватарки хранятся прямо в базе как сжатое изображение (клиент сам
  обрезает фото в квадрат и уменьшает перед отправкой на сервер).
- Пароли хранятся в базе в хэшированном виде (bcrypt), не в открытом тексте.
- Вход выдаёт JWT-токен на 30 дней, хранится в браузере (localStorage).
- Список пользователей и история переписки видны всем зарегистрированным.
- Новые сообщения, а также смена ника/аватарки, долетают мгновенно через
  WebSocket, если собеседники онлайн; иначе видны при следующем заходе.
- Интерфейс адаптируется под телефон: список чатов и переписка показываются
  по очереди (как в мобильном Telegram), на компьютере — как обычно, в
  три колонки.

## Двухфакторная аутентификация (2FA)
- Включается в **Настройки → Безопасность → Двухфакторная аутентификация**.
- Используется TOTP (как в Google Authenticator, Aegis, 1Password) —
  ничего, кроме приложения-аутентификатора, не требуется (ни почты, ни SMS).
- При включении показывается секретный ключ: добавьте его в приложение
  вручную («ввести ключ настройки») и подтвердите 6-значным кодом.
- После включения при каждом входе, кроме имени и пароля, попросят код из
  приложения. Отключить 2FA тоже можно только с действующим кодом.
- Реализовано на встроенном `crypto` (RFC 6238), без внешних зависимостей;
  секрет хранится в БД в колонке `totp_secret`.

## HTTPS / SSL-сертификаты
- Положите файлы сертификата и ключа в папку `certs/` (`cert.pem` и `key.pem`,
  при необходимости `ca.pem`) — сервер автоматически поднимется по HTTPS,
  а WebSocket заработает по `wss://`. Подробности — в `certs/README.md`.
- Пути можно переопределить переменными окружения `SSL_CERT_PATH`,
  `SSL_KEY_PATH`, `SSL_CA_PATH`.
- Если файлов нет — как и раньше, работает по обычному HTTP.
